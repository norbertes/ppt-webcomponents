Custom elements

Prototypowanie oraz rozszerzanie elementów 


document.registerElement('news-stream', {
  prototype: Object.create(HTMLElement.prototype)
});
1) registerElement posiada dwa argumenty, pierwszy to nazwa elementu, który tworzymy, drugim jest opcjonalny to prototyp, pozwalający na rozszerzenie funkcjonalności. Jeśli nie podamy 2giego argumentu, nasz customelement będzie posiadać HTMLElement jako prototyp. 2) możemy również użyć dziedziczenia po szczególnym rodzaju elementu HTML, dzięki temu możemy zmodyfikować, lub rozszerzyć sposób działania standardowych znaczników. Zeby utworzyć element rozszerzający element Form, nasz nowy element musi również dziedziczyć prototyp tego elementu.

Shadow DOM

Shadow DOM


var host = document.createElement('div');
var root = document.createElement('p');
root.innerHTML = 'Hejho!';
host.createShadowRoot().appendChild(root);
document.body.appendChild(host);
Shadow Host - element na którym wołamy createShadowRoot(). Jedyna część widoczna dla użytkownika, w której user może zadeklarować potrzebne wnętrzene informacje. Shadow Root - fragment dokumentu zwrócony przez createShadowRoot(). Ten element, oraz wszystkie jego dzieci są niewidoczne dla użytkownika, ale przeglądarka wyrenderuje jego całą zawartość. Shadow Boundary - każdy element wchodzący w skład shadow roota jest schowany za tą barierą. Blokuje ona przed obustronnym wyciekiem CSS oraz JS.

Shadow DOM

CSS

  • ::shadow (nazwa-componentu::shadow h1)
  • /deep/ (nazwa-componentu /deep/ h1)
  • :host (:host)
  • :host-context (:host-context(.parent-class))
  • ::content (::content h1)
  • :unresolved
Wszystkie selektory są dostępne w każdej przegląradce (chrome, safari, mozilla, opera) oraz IE od wersji 10. ::shadow - elementy na 1 poziomie zagnieżdżenia (gdzie poziom zagnieżdżenia to poziom webkomponentocepcji) /deep/ - elementy na dowolnym poziomie zagnieżdżenia :host - zaznacza element hosta (element na którym wołamy createShadowRoot(). Jedyna część widoczna dla użytkownika) :host-context - zaznacza hosta matchując parent element hosta ::content - zaznacza node'a wewnatrz komponentu :unresolved - posiadają ją elementy zainicjowane, ale nie zarejestrowane przy pomocy registerElement - ma słabe wsparcie. Służy do FOUC (Flash of Unstyled Content.) prevention, czyli wyświetlenia zasobu przez przeglądarkę przez załadowaniem zewnętrznego CSS'a

Webcomponents vs iFrame

Tylko dlaczego, skoro nie niesie to żadnych benefitów, a same przeszkody?

Webcomponents vs iFrame

Różnice

  • iframe jest kompletnym, oddzielnym dokumentem
  • dostęp do danych wewnątrz ramki jest niefajny (oddzielny kontekst, trudny dostęp do danych)
  • większa ilość tego samego komponentu na stronie = kilka iframeów o różnych urlach i danych
  • przy użyciu ramek - brak semantyki oraz spory overhead
